## 1. 软件概述

### 1.1 软件简介

《点滴清单》是一款基于Django框架开发的跨平台提醒管理工具，支持Web端、移动端和桌面端访问。该系统提供了多种类型的提醒功能，包括生日提醒、节日提醒、闹钟提醒和待办事项等，支持农历和阳历切换，并具备灵活的重复提醒机制。

### 1.2 主要功能特性

- 支持多种提醒类型：生日提醒、节日提醒、闹钟提醒、待办事项
- 支持农历与阳历提醒设置
- 灵活的重复提醒机制：每日、每周、每月、每年
- 用户认证与权限管理
- 实时提醒通知
- 提醒统计与管理
- Web端和桌面端双平台支持

### 1.3 技术架构

- 后端：Django REST Framework
- 数据库：SQLite3
- 前端：HTML/CSS/JavaScript + Bootstrap
- 桌面端：PyWebView
- 移动端：Android原生应用

## 3. 安装部署

### 3.1 服务器端安装

1. 下载源代码包
2. 安装Python依赖：
   ```bash
   pip install -r requirements.txt
   ```
3. 初始化数据库：
   ```bash
   python manage.py migrate
   ```
4. 创建超级用户：
   ```bash
   python manage.py createsuperuser
   ```
5. 启动服务器：
   ```bash
   python manage.py runserver 8000
   ```

### 3.2 桌面端安装
1. 运行桌面应用：
   ```bash
   python desktop_app.py
   ```
2. 或运行打包的应用程序：`dist/点滴清单.exe`

## 4. 功能说明

### 4.1 用户管理
#### 4.1.1 用户登录
- 打开桌面应用或访问Web端
- 输入用户名和密码
- 点击"登录"按钮完成身份验证

#### 4.1.2 用户注册
- 系统支持通过Web管理后台创建用户
- 管理员可使用Django Admin进行用户管理

### 4.2 提醒管理
#### 4.2.1 创建提醒
1. 登录系统后进入提醒管理页面
2. 点击"新建提醒"按钮
3. 填写以下信息：
   - 提醒类型：选择生日、节日、闹钟或待办事项
   - 标题：提醒的标题
   - 描述：提醒的详细说明
   - 日历类型：选择阳历或农历
   - 时间设置：设置具体的月份、日期、小时和分钟
   - 重复类型：选择不循环、每日、每周、每月或每年
   - 截止日期：可选设置提醒的结束时间

#### 4.2.2 编辑提醒
1. 在提醒列表中找到需要编辑的提醒
2. 点击"编辑"按钮
3. 修改相关信息后保存

#### 4.2.3 删除提醒
1. 在提醒列表中找到需要删除的提醒
2. 点击"删除"按钮
3. 确认删除操作

#### 4.2.4 确认提醒
- 当收到提醒通知时，可点击"确认"按钮标记已完成
- 系统会根据重复设置计算下次提醒时间

#### 4.2.5 推迟提醒
- 收到提醒后，可以选择"推迟"功能
- 设置推迟分钟数，提醒将在指定分钟后再次触发

### 4.3 提醒类型详解
#### 4.3.1 生日提醒
- 用于纪念个人生日
- 支持农历生日设置
- 可设置年度重复

#### 4.3.2 节日提醒
- 用于重要节日提醒
- 支持农历节日（如春节、中秋节等）
- 可设置年度重复

#### 4.3.3 闹钟提醒
- 用于日常定时提醒
- 支持一次性或重复提醒
- 可设置精确到分钟的时间

#### 4.3.4 待办事项
- 用于任务管理和提醒
- 支持单次或重复任务
- 可设置描述和优先级

### 4.4 统计功能
- 总提醒数量统计
- 活跃提醒数量统计
- 待处理提醒数量统计
- 按类型分类统计

## 7. API接口说明

### 7.1 认证接口
- `POST /api-token-auth/` - 用户认证获取token

### 7.2 提醒管理接口
- `GET /api/reminders/` - 获取用户所有提醒
- `POST /api/reminders/` - 创建新提醒
- `PUT /api/reminders/{id}/` - 更新提醒
- `DELETE /api/reminders/{id}/` - 删除提醒
- `GET /api/reminders/pending/` - 获取待处理提醒
- `POST /api/reminders/{id}/confirm/` - 确认提醒
- `POST /api/reminders/{id}/snooze/` - 推迟提醒
- `GET /api/reminders/statistics/` - 获取统计数据

## 8. 故障排除

### 8.1 常见问题
Q: 无法登录系统
A: 检查用户名密码是否正确，确认服务器是否正常运行

Q: 提醒没有按时触发
A: 检查服务器时间设置，确认后台任务是否正常运行

Q: 农历日期计算错误
A: 确认lunardate库是否正确安装和配置

### 8.2 错误处理
- 系统会记录详细的错误日志
- 日志文件位于 `logs/` 目录下
- 可通过日志分析系统运行状态
